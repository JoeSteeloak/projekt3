Processen - SASS

NodeJs och NPM

Jag installerade den senaste versionen av Node.js från deras hemsida. Sen startade jag ett nytt projekt från kommandotolken med kommandot npm init och accepterade standardinställningarna. Efter det installerade jag Parcel med kommandot npm install parcel --save-dev vilket såg till att det var ett utvecklingsberoende, så det bara är med under utvecklingen och inte publiceringen. Jag skapade också en .gitignore-fil i mitt projekt med node_modules, dist, och .parcel-cache eftersom NPM skapar de på nytt ändå när jag publicerar hemsidan.

Parcel och Sharp

Parcel hjälper att minifiera och transpilera filerna i mitt projekt och när jag la till bilder på min hemsida installerades Sharp som automatiskt optimerar bilderna med filtyp, storlek och kvalitet. Med hjälp av koden "browserslist": "> 0.5%, last 2 versions, not dead" gör jag också mitt projekt bakåtkompatibelt för att den ska fungera i så många webbläsare som möjligt. Jag tog bort defer på mina js-länkar i html-koden och la till type="module för att de skulle bli transpilerade.

Versionshantering

I mitt projekt skapar jag en mapp som jag döper till src där själva hemsidan byggs och som skiljer sig från dist som skapas automatiskt när filerna minifierats för publicering. Jag gjorde mitt projekt till ett repository på GitHub och skapade en dev-gren som jag jobbade aktivt på projektet från, med regelbundna commits, och som jag sammanfogade (merge) med main-grenen när jag ansåg den redo.

Publicering

Jag laddar upp projektet till Netlify genom GitHub och Netlify sköter publiceringen.
